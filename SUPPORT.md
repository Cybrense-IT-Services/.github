# Support

## Questions and Setup Help

Start with the target repository's README, documentation, and troubleshooting
guide. If the answer is not there, open a GitHub issue in that repository and
include:

- the project version or commit;
- operating system and relevant runtime versions;
- configuration details with all secrets removed;
- the expected and actual behavior;
- logs or screenshots that do not contain sensitive data.

## Bugs and Feature Requests

Use the issue templates in the relevant public repository. Search for an
existing issue first and add useful context there when one already exists.

## Security Issues

Never report a vulnerability in a public issue. Follow our
[Security Policy](SECURITY.md).
