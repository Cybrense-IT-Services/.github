# Contributing to Cybrense Projects

Thank you for helping improve Cybrense open-source projects. We value focused
changes, clear communication, and software that is secure and maintainable.

## Before You Start

1. Read the target repository's README and local `CONTRIBUTING.md`, if present.
   Repository-specific guidance takes precedence over this shared guide.
2. Search existing issues and pull requests before opening a new one.
3. Open an issue before starting a large feature, architectural change, or
   breaking change so the direction can be discussed early.
4. Do not disclose security vulnerabilities in public issues. Follow our
   [security policy](SECURITY.md) instead.

## Making a Change

- Keep each pull request limited to one clear purpose.
- Match the existing code style and project structure.
- Add or update tests for behavior changes.
- Update documentation when setup, configuration, or user-facing behavior changes.
- Never commit credentials, personal data, production configuration, or generated secrets.
- Use clear commit messages that explain the intent of the change.

## Pull Request Checklist

Before requesting review, confirm that:

- the project builds and its relevant checks pass;
- new behavior is covered by appropriate tests;
- documentation is accurate;
- the change does not introduce secrets or sensitive data;
- the pull request explains what changed, why, and how it was verified.

## Review

Maintainers may request changes for correctness, security, maintainability,
scope, or documentation. Reviews are collaborative: explain tradeoffs and keep
technical discussion respectful and evidence-based.

By participating, you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
